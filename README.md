# lightforge-docs

Documentation and blog platform for [lightforge.dev](https://lightforge.dev).

- **Docs:** [docs.lightforge.dev](https://docs.lightforge.dev)
- **Blog:** [lightforge.dev/blog](https://lightforge.dev/blog)

Built with [Astro Starlight](https://starlight.astro.build/). MIT.

## Layout

```
apps/
  docs/         # Astro Starlight, base: '/'  → docs.lightforge.dev
  blog/         # Astro Starlight, base: '/blog' → lightforge.dev/blog (edge-routed)
packages/
  theme/        # Shared brand tokens (Cleo)
.github/
  workflows/ci.yml          # Build + sanctioned-components lint
  PULL_REQUEST_TEMPLATE.md  # Editorial gates
  CODEOWNERS
```

## Develop

```sh
npm install
npm run dev:docs   # http://localhost:4321
npm run dev:blog   # http://localhost:4321/blog
```

## Build

```sh
npm run build:docs   # → apps/docs/dist
npm run build:blog   # → apps/blog/dist (paths under /blog)
```

## Deployment

Both apps deploy to **Cloudflare Pages** as separate projects, auto-built on push to `main`. PR previews enabled.

- `lightforge-docs` project → `docs.lightforge.dev` (CNAME)
- `lightforge-blog` project → routed by Cloudflare Page/Transform Rule from `lightforge.dev/blog/*`. Phoenix never sees `/blog/*` traffic.

Infra owned by Jasper.

## Editorial workflow

Every PR runs:

- MDX build (both apps via matrix)
- Sanctioned-components lint (no raw `<iframe>` / `<script>` / `<video>` / `<audio>` / `<embed>` / `<object>` in authored content)
- No-absolute-root-links smoke test on the blog build

Two human/agent gates per PR (see `.github/PULL_REQUEST_TEMPLATE.md`):

1. **Claim-evidence check** — Cleo. References `marketing/public-claims-standard.md`.
2. **Owner approval** — Aria, for commitment-bearing copy.

CODEOWNERS routes reviewers automatically.
